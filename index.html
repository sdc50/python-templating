Overview of Templating Systems

What is Templating?

A template just a text file. A template contains templating code (i.e. variables and/or logic expressions), which get replaced with other data (usually stored in a database) when the template is rendered. For web templating is normally used with HTML files. The rendered template is a finished HTML page that can be served to a client.

Templates can be compared to a form letter. A web templating system uses the "form letter" to either generate a large number in static pages in advance, or to dynamically produce web pages on demand.

Templates are rendared using a template engine. Python templating systems are used within Python web frameworks. A framework may have its own templating system (like Django), or may allow configuration to work with a number of templating systems (like Pyramid).

Features of Templating Systems

  • Variables
  • Filters
  • Logic Expressions
  • Inheritance

Variables

Variables are the simplest and most fundamental aspect of templates. A template variable is simply a value that is passed to the template. The value is then substituded wherever the variable name is found.

Filters

Filters are a way of modifying variables. Filters can be used to change the format of a variable, or to access its attributes.

Expressions

Expressions are great

This is some discussion about whether templates should contain any logic. In a traditional model-view-controller architechture the view should be independant of buisness logic. However, there is some logic which is inherent to the view, and templating system which provides syntax for control statements can be very userful. The documentation for Jinja2 sees it this way:

"Without a doubt you should try to remove as much logic from templates as possible. But templates without any logic mean that you have to do all the processing in the code which is boring and stupid. . . So some amount of logic is required in templates to keep everyone happy."
